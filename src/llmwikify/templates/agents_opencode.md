## Overview

This is a knowledge base powered by [llmwikify](https://github.com/sn0wfree/llmwikify).
The LLM reads raw sources and incrementally builds a persistent, interlinked wiki.

- **Sources**: `raw/` directory
- **Wiki**: `wiki/` (LLM-maintained markdown pages)
- **Schema**: `wiki.md` ‚Äî **read this before editing the wiki**

## Core Rules

1. **Read `wiki.md` first** ‚Äî it contains all page naming conventions, linking standards, source citation formats, and domain-specific templates
2. **Always use wiki tools** ‚Äî never modify wiki files directly
3. **Cite sources** ‚Äî every claim should be traceable to a file in `raw/`
4. **Use `[[wikilinks]]`** for cross-references between wiki pages
5. **Do not create duplicate pages** ‚Äî merge new info into existing pages
6. **Query answers go to sink** when a similar page exists ‚Äî use `wiki_synthesize` with default `merge_or_replace="sink"`
7. **Write graph relations** ‚Äî use `wiki_graph(action="write")` to maintain the knowledge graph alongside pages
8. **Track claims** ‚Äî separate factual claims from interpretation; use `wiki/claims/` pages with supporting/contradicting evidence

## Ingest Workflow

1. Call `wiki_ingest(source="raw/{file}")` ‚Äî returns full content, detected entities, and relations
2. Review the response for `detected_entities`, `detected_relations`, and `detected_claims`
3. Create appropriate wiki pages using `wiki_write_page`:
   - Always: `wiki/sources/{slug}.md`
   - If entities detected: `wiki/entities/{name}.md`
   - If concepts detected: `wiki/concepts/{name}.md`
   - If claims identified: `wiki/claims/{slug}.md`
4. Write graph relations: `wiki_graph(action="write", relations=[...])`
5. Add `[[wikilinks]]` between related pages
6. Call `wiki_log` to record what was done

## Knowledge Base Management

You have two ways to operate the wiki:

**MCP mode** (structured JSON):
- MCP tools are auto-configured in `opencode.json`
- Use `wiki_search`, `wiki_ingest`, `wiki_write_page`, etc.
- Better error handling and structured responses

**CLI mode** (via Bash):
- Use `llmwikify` CLI commands directly
- Skill content auto-loaded via `opencode.json` instructions
- See `.agents/skills/llmwikify/SKILL.md` for full details

**Key CLI commands:**
- `llmwikify ingest raw/file.md [--smart]` ‚Äî Extract and ingest source
- `llmwikify search "query"` ‚Äî Full-text search
- `llmwikify write_page "Name" --content "..."` ‚Äî Create/edit pages
- `llmwikify read_page "Name"` ‚Äî Read page content
- `llmwikify lint` ‚Äî Health check
- `llmwikify status` ‚Äî Wiki status summary
- `llmwikify batch raw/dir/` ‚Äî Batch ingest
- `llmwikify graph-query stats` ‚Äî Graph statistics

Choose whichever works best in your current context.
