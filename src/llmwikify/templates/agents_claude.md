## Context

This project uses llmwikify to maintain a persistent, LLM-written knowledge base.
Raw source files live in `raw/` directory.
The wiki is in `wiki/` and is owned entirely by the LLM.

**CRITICAL: Read `wiki.md` before making any wiki edits.** It contains:
- Page naming conventions
- Linking standards (`[[wikilinks]]`)
- Source citation formats
- Domain-specific templates
- Workflows for ingest, query, and maintenance

## MCP Setup

The llmwikify MCP server is configured in `.mcp.json`. When you start Claude Code in this directory, the tools load automatically. Available tools: `wiki_init`, `wiki_ingest`, `wiki_write_page`, `wiki_read_page`, `wiki_search`, `wiki_synthesize`, `wiki_lint`, `wiki_status`, `wiki_build_index`, `wiki_sink_status`, `wiki_references`, `wiki_read_schema`, `wiki_update_schema`, `wiki_log`, `wiki_graph`, `wiki_graph_analyze`.

## Knowledge Base Management

You have two ways to operate the wiki:

**MCP mode** (structured JSON):
- MCP tools are auto-configured in `.mcp.json`
- Use `wiki_search`, `wiki_ingest`, `wiki_write_page`, etc.

**CLI mode** (via Bash):
- Use `llmwikify` CLI commands directly
- Skill content auto-loaded via instructions
- See `.agents/skills/llmwikify/SKILL.md` for full details

**Key CLI commands:**
- `llmwikify ingest raw/file.md [--smart]`
- `llmwikify search "query"`
- `llmwikify write_page "Name" --content "..."`
- `llmwikify lint`
- `llmwikify status`
- `llmwikify batch raw/dir/`
- `llmwikify graph-query stats`

Choose whichever works best in your current context.
See `.agents/skills/llmwikify/resources/cli-reference.md` for full CLI reference.

## Rules

- Read `wiki.md` first for conventions and templates
- Always use wiki MCP tools — never edit wiki files directly
- Cite raw sources for every claim (use `[Source](raw/path)` format)
- Use `[[wikilinks]]` between wiki pages
- When a similar query page exists, answers go to the sink — review and merge during lint
