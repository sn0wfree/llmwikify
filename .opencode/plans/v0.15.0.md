## 1. Background

### 1.1 Principle Gaps (from `docs/LLM_WIKI_PRINCIPLES.md`)

| Principle | Current State | Gap |
|-----------|---------------|-----|
| Ingest: "LLM reads source, discusses key takeaways" | Returns only file path | LLM lacks file metadata (type, size, images) |
| Lint: "contradictions between pages" | Missing | No detection at all |
| Lint: "stale claims superseded by newer sources" | Missing | No detection at all |
| Lint: "missing cross-references" | broken_links only | Unlinked concept mentions not detected |

### 1.2 Design Philosophy

**Pure Tool Principle**: System provides data, LLM makes judgments. Never auto-decide what counts as a "contradiction" or "data gap" — those require domain knowledge.

---

## 2. Design Decisions

### 2.1 Ingest Metadata Enhancement

**Decision**: Return rich metadata, **NOT summaries**.

**Rationale**:
- Summary extraction = "thinking" = LLM's job
- Metadata = "facts" = system's job
- Aligns with zero domain assumption

**New Return Structure**:
```json
{
    "status": "ok",
    "file": "raw/article.md",
    "file_type": "markdown",
    "file_size": 15432,
    "word_count": 2340,
    "has_images": true,
    "text_extracted": true,
    "content_preview": "First 200 chars...",
    "message": "Source ingested. Read the file to extract key takeaways."
}
```

### 2.2 Lint Clue-Based Detection

**Decision**: Three clue types, LLM makes final judgment.

| Clue Type | Category | Detection Logic | Threshold | Max |
|-----------|----------|-----------------|-----------|-----|
| `dated_claim` | critical | Page mentions year N, latest source is year N+3+ | >= 3 years | 3 |
| `topic_overlap` | informational | Jaccard overlap between Query: pages | >= 85% | 2 |
| `missing_cross_ref` | informational | Concept mentioned in >= 2 pages without wikilink | >= 2 pages | 3 |

**Rejected Detections**:
- `orphan_concept` — requires domain judgment about what "deserves" a page
- Auto contradiction detection — violates zero domain assumption
- Data gap detection — too vague, depends on user needs

### 2.3 Hint Fatigue Prevention

**Strategy**: Two-tier classification + hard limits.

```json
{
    "hints": {
        "critical": [        // max 3 items
            {"type": "dated_claim", ...}
        ],
        "informational": [   // max 5 items
            {"type": "topic_overlap", ...},
            {"type": "missing_cross_ref", ...}
        ]
    }
}
```

**Design**: `critical` hints demand attention; `informational` hints are optional context. Total max 8 hints per lint pass.

```python
def _detect_dated_claims(self) -> List[dict]:
    """Find year mentions (2018-2023) that predate latest raw source by 3+ years."""

def _detect_query_page_overlap(self) -> List[dict]:
    """Find Query: pages with >=85% keyword Jaccard overlap."""

def _detect_missing_cross_refs(self) -> List[dict]:
    """Find concepts mentioned in 2+ pages but not wikilinked."""
```
