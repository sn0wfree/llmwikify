## What's New in v0.37.0

v0.37 unifies the three scattered ReAct loop implementations (ChatService, ResearchEngine,
AutoResearch) into a single `ReActEngine` framework. ChatService integrates through a new
`ChatReActBridge` adapter; ResearchEngine integrates via a config adapter
(`_build_react_config`).

**Before** (v0.36): 3 ReAct implementations, lifecycle hooks duplicated, cancel/timeout logic
fragmented.

**After** (v0.37): 1 `ReActEngine` + 2 adapters. All chat/research paths share:

- 13-step round structure (`pre_reason`, `reason`, `post_reason`, ..., `on_round_complete`)
- 7 lifecycle hooks (`on_session_start`, `on_round_start`, `on_round_complete`,
  `on_session_complete`, `on_error`, `on_timeout`, `on_cancel`)
- Standard event types (`reasoning`, `action`, `observation`, `round_complete`, `action_error`,
  `observation_error`, `phase`, `timeout`, `done`)
- `AbortSignal` cancellation
- Configurable timeout (default 300s)

## 1. The 3 New Modules

### 1.1 `ReActEngine` (`apps/chat/agent/react_engine.py`, 685L)

The unified ReAct framework. All ReAct-based flows eventually call `ReActEngine.run()`.

```python
from llmwikify.apps.chat.agent.react_engine import (
    ReActEngine, ReActConfig, SkillAction, SkillContext, SkillResult
)

config = ReActConfig(
    max_rounds=4,
    timeout_s=300.0,
    actions=[skill_action_1, skill_action_2, ...],
    hooks={
        "on_round_complete": my_hook,
        ...
    },
    reason=my_reason_callback,  # async (messages) -> AsyncIterator[event]
)

engine = ReActEngine(config)
async for event in engine.run(skill_context):
    if event["type"] == "reasoning": ...
    if event["type"] == "action": ...
    if event["type"] == "done": break
```

**Key responsibilities:**

- Round orchestration (13 steps)
- Lifecycle hook dispatch
- Timeout enforcement
- Cancel propagation via `AbortSignal` or `CancelToken`
- Action dispatch (with `_handle_action` for custom action types)
- Round-level decision (continue / done / fail)

**Test coverage:** 50+ unit tests in `tests/test_apps_chat_agent_react_engine.py`.

### 1.2 `ChatReActBridge` (`apps/chat/agent/chat_react.py`, 617L)

Adapter that lets `ChatService` use `ReActEngine` while preserving chat-specific behavior.

**Responsibilities:**

| Method | Purpose |
|---|---|
| `translate_react_event` | ReAct event → SSE event (message_delta, tool_call_start, ...) |
| `parse_text_tool_call` | Parse `[TOOL_CALL]...[/TOOL_CALL]` blocks |
| `aggregate_observations` | Merge parallel tool results |
| `inject_react_prompt` | Add ReAct instruction to system prompt |
| `truncate_message` | Regenerate flow: cut at user message |
| `retry_llm` | First-chunk retry (3x, exponential backoff) |
| `persist_thinking` | Snapshot thinking into context |

`ChatService` is wired to `ChatReActBridge` by default via `use_react_engine=True` in
`AgentService` composition.

**Test coverage:** 30+ unit tests in `tests/test_apps_chat_agent_chat_react_bridge.py`.

### 1.3 `text_mode_tool` (`apps/chat/agent/text_mode_tool.py`, 205L)

Shared parser for text-mode tool calls emitted by LLMs without native tool_calls support:

```
[TOOL_CALL] {tool => "wiki_read_page",
             args => { --page_name "overview" }} [/TOOL_CALL]
```

Used by both `ChatService` (via `_stream_preprocess`) and `ChatReActBridge`.

**Exports:** `TOOL_CALL_RE`, `parse_text_tool_call`, `parse_perl_args`, `TextModeParser`.

## 2. Refactors

### 2.1 `ChatService` default path (`agent_service.py`)

```python
self.chat_service = ChatService(
    self.data_dir,
    chat_db=self.app_db.chat,
    memory_manager=self.memory_manager,
    use_react_engine=True,  # v0.37 default; was implicitly False in v0.36
)
```

The `aask_with_tools` path is preserved as a fallback (`use_react_engine=False`). This is
verified by regression tests on both paths.

### 2.2 `ResearchEngine` uses `ReActEngine`

`apps/chat/engine.py:_react_loop` (the 6-step ResearchEngine loop) is replaced by
`_build_react_config()` which:

1. Maps the 6 ResearchEngine steps to 13 generic round steps
2. Wraps each ResearchEngine action as a `SkillAction`
3. Emits `EVENT_PHASE` events for domain semantics (e.g. "planning", "searching", "writing")
4. Lets `ReActEngine` handle timeout/cancel/lifecycle uniformly

This is a 416-line diff but the public API (`ResearchEngine.run_research`) is unchanged.

### 2.3 `service.py` imports

`service.py` adds imports for the text-mode parser (shared with the bridge).

## 3. New Event Types (ReAct path)

| Event | Source | Description |
|---|---|---|
| `reasoning` | ReActEngine | LLM chain-of-thought / thinking |
| `action` | ReActEngine | Tool invocation |
| `observation` | ReActEngine | Tool result |
| `round_complete` | ReActEngine | Round ended |
| `action_error` | ReActEngine | Tool call failed |
| `observation_error` | ReActEngine | Observation failed |
| `phase` | ResearchEngine adapter | Domain phase marker |
| `timeout` | ReActEngine | Total timeout exceeded |

Existing SSE events (`message_delta`, `tool_call_start/end/error`, `confirmation_required`,
`done`, `error`, `save_warning`, `timeout`) continue to be emitted by `ChatReActBridge`.

## 4. Lifecycle Hooks

All 7 hooks are implemented in `ReActEngine`:

| Hook | When | Use case |
|---|---|---|
| `on_session_start` | `run()` begins | Init DB row, allocate session |
| `on_round_start` | Each round begins | Per-round state reset |
| `on_round_complete` | Each round ends | Per-round cleanup |
| `on_session_complete` | `done` reached | Persist final state |
| `on_error` | Exception caught | Custom error reporting |
| `on_timeout` | Total timeout exceeded | Cleanup partial state |
| `on_cancel` | AbortSignal triggered | Cancel pending tools |

Hooks are passed via `ReActConfig.hooks` as async callables.

## 5. Migration

### 5.1 Caller code (unchanged)

`ChatService.chat()`, `ResearchEngine.run_research()`, etc. — public API unchanged.

### 5.2 New capabilities

- All chat flows now produce uniform `reasoning` events (v0.36 had scattered thinking snapshots).
- Research flows can be cancelled mid-run via `AbortSignal`.
- Research flows have explicit timeout events (was previously silent).

### 5.3 Internal changes

- `apps/chat/engine.py::_react_loop` is removed; replaced by `_build_react_config()`.
- `service.py::_stream_preprocess` delegates text-mode parsing to `text_mode_tool`.
- The old `framework/react.py` still works (backward compat) but will be deprecated in v0.38.

### 5.4 Deprecation path

`framework/react.py` is marked deprecated. Will be removed in v0.38.

## 10. Bridge Event Translation Map

`ChatReActBridge.translate_react_event` maps internal ReActEngine events to client-facing SSE
events. This is the contract the webui consumes.

| ReActEngine event | SSE event | Notes |
|---|---|---|
| `EVENT_REASONING` (delta) | `message_delta` | Concatenated into assistant content |
| `EVENT_REASONING` (snapshot) | `thinking` | Chain-of-thought preserved separately |
| `EVENT_ACTION` (start) | `tool_call_start` | Carries `id`, `name`, `arguments` |
| `EVENT_ACTION` (success) | `tool_call_end` | Carries `id`, `result` |
| `EVENT_ACTION_ERROR` | `tool_call_error` | Non-fatal; stream continues |
| `EVENT_OBSERVATION` | (folded into next `message_delta`) | Not emitted standalone |
| `EVENT_OBSERVATION_ERROR` | `tool_call_error` | Same event type as action error |
| `EVENT_PHASE` | `phase` | ResearchEngine domain marker |
| `EVENT_ROUND_COMPLETE` | (internal) | Not emitted to client |
| `EVENT_TIMEOUT` | `timeout` | 300s exceeded |
| `EVENT_DONE` | `done` | Stream complete |

Plus events NOT translated (client only sees SSE side):

| Internal | What it does |
|---|---|
| `EVENT_SESSION_START` | Trigger `on_session_start` hook |
| `EVENT_SESSION_COMPLETE` | Trigger `on_session_complete` hook |
| `EVENT_CANCEL` | Trigger `on_cancel` hook (after abort) |

Plus SSE-only events emitted by `ChatReActBridge` directly (not from ReActEngine):

| SSE event | Trigger |
|---|---|
| `session_created` | First event in stream |
| `save_warning` | DB write failed |
| `error` | Fatal error |
