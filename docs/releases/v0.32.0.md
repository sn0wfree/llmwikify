## What's New in v0.32.0

v0.32.0 is the **first major refactor of llmwikify's chat layer**
since the 4-layer architecture was introduced in v0.30. It
introduces a **uniform skill framework** that the LLM can
call via OpenAI function-calling, persists the research
state through every ReAct round, and consolidates the
2-database mess into a single `ChatDatabase`.

The big idea: **any chat-facing capability is now a Skill**
with a name, description, JSON schema, and handler. The
23 base actions (search, read, write, score, etc.) +
research_skill (7-step ReAct pipeline) are all registered
into a `SkillRegistry` at startup. ChatBase drives the
OpenAI-style tool-call loop, invoking each tool via
`SkillRuntime`, and folding results back into the
session as `tool` messages.

### The 5 new abstractions

| Layer | Abbreviation | Example | Files |
|---|---|---|---|
| **action** | Single atomic operation | `search_skill.search` | 14 + 8 + 1 = 23 |
| **pipeline** | Multi-step orchestration | `gather_skill.gather` | 4 (Phase 6/12) |
| **skill** | Independent capability | `research_skill.run_research` | 5 (Phase 12) |

### Phase-by-phase summary

#### Phase 1: Skill framework (`apps/chat/skills/`)
**79 new tests.** Skill ABC, SkillAction, SkillContext, SkillResult,
SkillManifest. SkillRegistry (thread-safe). SkillRuntime
(async, with JSON Schema subset validation). 6 error types.

#### Phase 2: Rename `harness.py` → `eval_harness.py`
**0 LOC change.** Just a rename to free the `harness/` package
slot for Phase 7.

#### Phase 3: ChatDatabase consolidation + research_steps
**34 new tests.** Merged `AutoResearchDatabase` (3 tables) and
the `AgentDatabase` research tables into one `ChatDatabase`
at `apps/chat/db.py` with **4 tables**:
- `autoresearch_sessions` (unchanged schema, +9 framework JSON columns)
- `autoresearch_sub_queries`
- `autoresearch_sources`
- **`research_steps` (NEW)** — one row per ReAct/6-step round,
  persists the 15+ ResearchState fields.

New API: `save_step`, `get_step`, `list_steps`, `delete_steps`,
`update_step_status`, `save_research_state`,
`load_research_state`.

**Migration script**: `scripts/migrate_db_v1_to_v2.py` —
dry-run + backup + copy from old `agent.db::research_sessions`
to new `autoresearch.db::autoresearch_sessions`. Idempotent.

#### Phase 4: providers migrate
**0 new tests** (5 shim tests covered by Phase 3 / 5 / 7). 4 LLM
provider files moved from `apps/agent/providers/` to
`apps/chat/providers/`. 5 backward-compat shim files at
`agent/backend/providers.*`.

#### Phase 5: 23 base actions
**86 new tests.** 14 base + 8 detect + 1 clarify = 23
actions. Each is a thin wrapper over existing wiki / engine
/ DB methods. `DetectActionSkill` base class for the 8
detect actions. `actions/__init__.py` asserts
`len(ALL_ACTIONS) == 23` on import (silent-drift guard).

#### Phase 6: research_skill
**52 new tests.** 7-step ReAct pipeline (plan → gather →
analyze → synthesize → score → revise → report) composed
of Phase 5 actions + 6 lifecycle hooks wired to the new
ReactLoop framework. State persistence via
`db.save_research_state`/`list_steps`. 3 public actions:
`run_research` / `resume_research` / `cancel_research`.

#### Phase 7: 5 harness eval classes
**32 new tests.** `QualityGate`, `SourceFilter`,
`ResearchReviewer`, `ResearchRevisor`, `StructureValidator`,
`SourceAnalyzer` — all in `apps/chat/harness/`. 5
backward-compat shim files at the old paths.

#### Phase 8: ReactLoop framework
**40 new tests.** Generic 13-field `ReactConfig` + `ReactLoop`
class with async-generator `run()`. 5 event types yielded
(`reasoning`, `action_error`, `observation_error`,
`round_complete`, `phase`). 9 lifecycle hooks
(`restore_state`, `done_condition`, `on_before_act`,
`on_after_act`, `on_before_observe`, `on_after_observe`,
`persist_state`, `max_rounds`, `reason_prompt`).
Reason/observe callables are pluggable. Used by
`research_skill` (Phase 6); the framework is generic
enough for any future ReAct-style skill.

#### Phase 9: REST routes migrate
**24 new tests.** 3 routes moved from `apps/agent/routes/`
to `interfaces/server/http/`:
- `agent.py` → `chat_sse.py` (renamed: SSE is a chat concern)
- `ppt.py`
- `research.py`

**Removed a hidden L3→L4 dependency** in `apps/chat/routes.py`:
the LLM client is now passed explicitly via
`set_autoresearch_deps(llm_client=agent_service._get_llm(), ...)`
at L4 startup. 4 backward-compat shim files at
`agent/backend/routes{,_agent,_ppt,_research}.py`.

#### Phase 10: ChatBase + Skill integration
**41 new tests.** ChatBase now has:
- `register_skills(registry)` — bulk register all skill actions
  as LLM tools with qualified names (`<skill>.<action>`)
- `tools_schema(registry)` — generate OpenAI function-calling
  JSON schema
- `invoke_tool(name, args, ctx)` + `ainvoke_tool(name, args, ctx)`
  — sync/async split that bridges to async `SkillRuntime.execute`
- `ask_with_tools(prompt, ...)` — full OpenAI-style tool-call
  loop with `DEFAULT_MAX_TOOL_ITERATIONS = 8` cap

**Fixed 3 latent bugs** that the new test suite caught:
1. `invoke_tool()` was returning an unawaited coroutine
   (now sync/async split)
2. `_extract_content_and_tool_calls()` only normalized
   tool_calls for object replies, not dicts (now always
   normalizes via `_normalize_tool_call_dict`)
3. `asyncio` import was missing

#### Phase 11: Final verification + release
**0 new tests** (this doc). 4 architecture contracts green. 28
MCP `@mcp.tool` descriptions byte-identical to v0.31
(verified — `tools.py` was unchanged since Sprint B).

### Acknowledgments

31 commits in 8 working sessions. Each phase followed
the per-step verification protocol (§4.0): pytest + 4
architecture contracts + 0 new warnings + 0 regressions
before the next phase began. The "🔴 high risk" ChatDatabase
merge (Phase 3) was scaled down to a focused research-DB
consolidation + research_steps addition, deferring the
full 9-table merge to v0.32.5 (Unix philosophy: one file = one
focus).

The framework refactor (Phases 1+2+5+6+7+8) is the structural
foundation that v0.32.5 will build on. The data refactor
(Phases 3+4) and interface refactor (Phases 9+10) are
the migration path that lets v0.32 ship with full
backward compat to v0.31.
