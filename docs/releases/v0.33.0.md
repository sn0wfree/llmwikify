## What's New in v0.33.0

v0.33.0 introduces the **5+1-service architecture** — the
final composition layer that ties together all the
infrastructure built in v0.32.0 → v0.32.5:

```
llm_service + ChatDatabase            =  ChatService
ChatService + SkillService           =  AgentService
            + HarnessService         =
            + MemoryManager          =
AgentService + WikiService           =  main entry point
```

**Before** (v0.32.5): 2 services + 2 deprecated wrappers.
**After** (v0.33.0): 5+1 services wired by composition root.

---

## 1. The 4 New Services

### 1.1 `MemoryManager` (`apps/chat/memory/`)

Unified memory base with **6 stores**:

| Store | Backing | Purpose |
|---|---|---|
| `ConversationStore` | `app_db.chat` (chat_messages) | Chat history |
| `KnowledgeStore` | `wiki.query_sink` | Wiki pages |
| `ContextStore` | `app_db.chat` (context_entries) | RAG chunks, tool results |
| `ReActStateStore` | `app_db.research` (research_steps) | Per-round state |
| `UserPreferenceStore` | JSON file | User settings |
| `MemoryIndex` | in-memory | Unified search |

- ~370 LOC
- 14 new tests (covers all 6 stores + unified search)

### 1.2 `SkillService` (`apps/chat/skills/service.py`)

Facade over the skill framework. Provides a single
injection point for `register_all()` and `execute()`:

```python
svc = SkillService()
svc.initialize()
svc.register_all()  # 23 base + 4 CRUD + 2 pipelines + research + wiki_query
result = svc.execute("search", "search_pages", {"query": "python"}, ctx)
```

- ~120 LOC
- 8 new tests (init, lazy registry, register_all, list/get, reset)

### 1.3 `HarnessService` (`apps/chat/harness/service.py`)

Facade over 6 eval primitives. Lazy-initializes each
on first access (cached):

```python
harness = HarnessService(config={}, wiki=wiki, llm=llm)
harness.quality_gate.gate(report)
harness.source_filter.filter(sources)
harness.structure_validator.validate(markdown)
harness.research_reviewer.review(report)
harness.research_revisor.revise(report, feedback)
harness.source_analyzer.analyze(sources)
```

- ~110 LOC
- 11 new tests (all 6 primitives + caching + LLM routing)

### 1.4 `AgentService` composition root (`apps/chat/agent/agent_service.py`)

The new **non-deprecated** entry point. Wires all 4 new
services + ChatService + WikiService + AppDatabase:

```python
agent = AgentService(wiki_registry, data_dir)
# All services accessible as attributes:
agent.app_db, agent.wiki_service, agent.chat_service,
agent.skill_service, agent.harness_service, agent.memory_manager
```

Backward compat: `agent.chat(...)`, `agent.run_dream(...)`,
`agent.list_notifications(...)` etc. all delegate to the
appropriate service.

- ~220 LOC
- 10 new tests (init, custom components, delegation, backward compat)

## 3. Migration

### 3.1 New code style (preferred)

```python
# New (v0.33.0)
from llmwikify.apps.chat.agent.agent_service import AgentService

agent = AgentService(wiki_registry, data_dir)
agent.memory_manager.conversation.add(session_id, "user", "hello")
agent.harness_service.quality_gate.gate(report)
```

### 3.2 Backward compat (still works)

```python
# Old (v0.32.5)
from llmwikify.apps.agent.core.service import AgentService
agent = AgentService(wiki_registry, data_dir)  # deprecated wrapper
```

Both work. The old wrapper is kept and will be removed in
v0.34.0. Deprecation warnings are emitted on construction.
