# Onboarding Improvements (v0.38+)

> **TL;DR**: Three commits (`302b15c`, `0b6457c`, `2fef73e`) systematically fixed the
> top 10 user-facing barriers from a fresh `git clone` to a working knowledge base.
> This document explains what was broken, what changed, and the design rationale.

---

## Why this document exists

A user audit of llmwikify (2026-07) found that someone who just `git clone`d the
repository would hit **3 documented failure paths** before they could do anything
useful — despite the underlying code being **deeply functional** (32 CLI commands,
8 runnable playbooks, FTS5 + graph + MCP all working). The gap was not in capability
but in **discoverability and first-run guidance**.

The audit surfaced 10 specific barriers. This document lists them, the fix, and
how to verify.

---

## Barriers fixed

### 1. ❌ `wiki.md` showed "v0.11.0" forever

**Symptom**: `llmwikify init` wrote `wiki.md` with a header reading
`> Generated by llmwikify v0.11.0` — even on v0.38.0. Made the project look
unmaintained to any LLM reading the schema.

**Root cause**: `WikiUtilityMixin._get_version()` did
`from .. import __version__` (kernel/wiki doesn't have `__version__`), fell
back to hardcoded `"0.11.0"`.

**Fix** (`302b15c`): `kernel/wiki/mixins/core/utility.py:54-60` — import from
`llmwikify` directly.

**Verify**: `head -3 wiki.md` shows `> Generated by llmwikify v0.38.0`.

---

### 2. ❌ Quick Start suggested `pip install llmwikify` (insufficient)

**Symptom**: README said `pip install llmwikify`, then `llmwikify ingest document.pdf`
— but minimal install can't handle PDF. User got ImportError or markdown fallback.

**Root cause**: PDF ingest needs `[extractors]` extra (MarkItDown + pymupdf),
but README didn't mention it.

**Fix** (`302b15c`): `README.md:64` — changed to
`pip install 'llmwikify[extractors,web]'`.

---

### 3. ❌ `examples/04_chat_sse_client/play.py` was missing

**Symptom**: `examples/README.md` told users to run
`cd 04_chat_sse_client && python play.py` — but `play.py` didn't exist.

**Fix** (`302b15c`): Created the missing script. Now runs end-to-end:
connect → send message → print 8 SSE event types (`session_created`, `reasoning`,
`phase`, `tool_call`, `confirmation_required`, `save_warning`, `stream_end`, `error`).

**Verify**:
```bash
cd examples/04_chat_sse_client
# Need running server: llmwikify serve --web --port 8765 --auth-token mysecret
python play.py  # uses defaults: localhost:8765, mysecret, "列出 wiki 里所有的页面"
```

### 4. ❌ Test count badge showed "5900+" (unverifiable)

**Symptom**: README badge said "5900+ passed", but CONTRIBUTING.md said
"3100+ tests" — no way to verify either.

**Fix** (`302b15c`): Ran `pytest --collect-only -q` (actual: 6192 tests).
Updated to "6100+ passed" in README, ARCHITECTURE, CONTRIBUTING. Slight undercount

**Verify**: `pytest --collect-only -q | tail -1` → should show ~6000+ tests.

### 5. ❌ Quick Start only showed 4 commands

**Symptom**: User couldn't discover `write_page`, `search`, `read_page`,
`build-index`. They had to dig into `TUTORIAL.md` or source code.

**Fix** (`302b15c`): Expanded Quick Start to 7 commands:
`init → write_page → ingest → build-index → search → read_page → status → serve`.

### 6. ❌ CLI Reference table had 12 of 32 commands

**Symptom**: `analyze-source`, `graph-query`, `community-detect`, `db`,
`knowledge-gaps`, `qmd`, `watch`, `auth` not in README's CLI table.

**Fix** (`302b15c`): Expanded table to all 32 commands, grouped by 9 categories

### 7. ❌ `~/.llmwikify/llmwikify.json` not documented

**Symptom**: LLM features need this file but README never mentioned it. User with
`OPENAI_API_KEY` exported got `LLM is not enabled. Set llm.enabled=true in config.`
with no path back.

**Root cause**: Three places documented the file separately (CONFIGURATION_GUIDE had
one sentence; TUTORIAL had troubleshooting bullets; designs/ had fragments).
None were discoverable.

**Fix** (`0b6457c`): Added dedicated section to
[`docs/CONFIGURATION_GUIDE.md`](./CONFIGURATION_GUIDE.md) with examples for
OpenAI / Anthropic / Minimax / OpenAI-compatible custom endpoints.

**Fix** (`2fef73e`): `llmwikify init-llm` now auto-detects provider from
`OPENAI_API_KEY` / `ANTHROPIC_API_KEY` / `MINIMAX_API_KEY` and creates the
config with one command.

---

### 8. ❌ No way to verify LLM key actually works

**Symptom**: User runs `llmwikify analyze-source` after setting up config.
Gets cryptic error. Is it the key? The model? The network?

**Fix** (`2fef73e`): `llmwikify doctor` now does **actual LLM connectivity test**.
Sends `"Say hi"` with `max_tokens=5`, 5-second timeout. Output:
```
✅ minimax/minimax-M3 — responded in 1.2s
```

Errors are categorized: `auth error (HTTP 401)`, `network error: ConnectError`,
etc.

---

### 9. ❌ `serve --web` shows blank page with no warning

**Symptom**: `llmwikify serve --web` starts but browser shows blank. User thinks
WebUI is broken. Real cause: `ui/webui/dist/` bundle not built (gitignored, not
in PyPI wheel).

**Fix** (`2fef73e`): `serve.py` now checks for `ui/webui/dist/index.html` before
starting. If missing, prints:
```
⚠️  WebUI bundle not built — the page may be blank.
   Fix: cd ui/webui && pnpm install && pnpm build
```

---

### 10. ❌ TUTORIAL.md referenced non-existent §0

**Symptom**: `examples/README.md` linked to `TUTORIAL.md#0-prerequisites-install-matrix--decision-tree` — that section didn't exist.

**Fix** (`2fef73e`): Replaced the broken link with a one-line Prerequisites summary:
"Python 3.10+, `pip install 'llmwikify[extractors,web]'`, optional LLM key".

## New tools

### `llmwikify init-llm` (and `--llm` on `init`)

Three ways to set up LLM config:

```bash
# Option 1: One-shot during init
export OPENAI_API_KEY=sk-...
llmwikify init --llm

# Option 2: Standalone
llmwikify init-llm
llmwikify init-llm --provider openai --api-key sk-...
llmwikify init-llm --base-url https://api.deepseek.com/v1  # OpenAI-compatible

# Option 3: Interactive (prompts on first init if no config)
llmwikify init
# No config detected. Set one up now? [y/N]: y
```

**Non-interactive for CI**:
```bash
llmwikify init --llm --no-llm-prompt --llm-overwrite
```

`init-llm` retained as standalone command for backward compat. Both call shared
`create_llm_config()` function in `init_llm_cmd.py`.

---

### `llmwikify doctor` (enhanced)

```bash
llmwikify doctor                     # 9 checks, including LLM API call
llmwikify doctor --skip-llm          # Skip API call (~1s)
llmwikify doctor --wiki-root /path   # Check specific wiki
llmwikify doctor --json              # JSON for CI/scripts
```

| # | Check | What it verifies |
|---|-------|------------------|
| 1 | Config | `~/.llmwikify/llmwikify.json` exists, parseable, has valid `api_key` |
| 2 | Python | Version >= 3.10 |
| 3 | Core deps | llmwikify, yaml, duckdb, jinja2 |
| 4 | Optional extras | fastapi, fastmcp, watchdog, networkx, markitdown, tiktoken, httpx |
| 5 | **LLM connectivity** | **Actual API call** (5s timeout) with `"Say hi"` |
| 6 | **Wiki directory** | wiki.md, .llmwikify.db, index.md, raw/ present |
| 7 | **Permissions** | `~/.llmwikify/` and wiki root are writable |
| 8 | WebUI bundle | `ui/webui/dist/index.html` exists |
| 9 | Server | `GET /api/health` returns 200 |

**Exit codes**: 0 (all pass), 1 (failed), 2 (config missing).

## Design decisions

### Why merge `init-llm` into `init` (option B)?

Four options were considered:

| Option | Verdict |
|--------|---------|
| A. Keep separate | Clean separation, but user needs 2 commands |
| **B. `init --llm` flag (chosen)** | **Single entry point, init-llm retained as alias** |
| C. Auto-prompt on first init | Most friendly, but changes existing behavior |
| D. Both B and C | More complex, marginal benefit |

**Why B**: User can run `llmwikify init` and **get prompted** to set up LLM
(defaults to N, doesn't break existing flows). User can also pass `--llm`
explicitly for one-shot setup. `init-llm` kept as alias for scripts.

### Why LLM connectivity test by default?

Adding a `--skip-llm` flag means **the slow case is opt-out, not opt-in**.
This trades 5 seconds for catching broken keys (which is the #1 user frustration
in the audit). CI users can pass `--skip-llm` to opt out.

### Why JSON output mode?

CI/CD integration was a recurring theme in the audit. `jq` filtering on
`summary.failed == 0` is much more reliable than parsing human-readable output.
JSON also enables monitoring dashboards.

## What was NOT changed

- **WebUI bundle not in PyPI wheel** (intentional — too heavy, builds on demand)
- **`pip install llmwikify` minimal install still works** for offline-only use
- **Existing 32 CLI commands unchanged** (only docs and config UX improved)
- **`init-llm` kept as alias** (not deprecated; backward compat)
