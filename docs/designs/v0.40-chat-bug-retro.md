## 真实根因（精确到字节）

`src/llmwikify/apps/chat/agent/service.py` 第 22-23 行附近缺 `import asyncio`。

第 253 行调用 `abort_event = asyncio.Event()` → `NameError: name 'asyncio' is not defined`。

服务端返回 200 OK + 空 SSE body。前端 SSE parser 静默吞掉空数据 → 用户看到"按了发送没反应"。

**这个 bug 整个 11 个 commit 期间一直没被发现**，因为：
- pytest 跑的是 mock，不发真实 HTTP 请求
- `tsc --noEmit` 不查 Python
- `vite dev` 宽容地打包，TypeScript 错不阻断
- 服务端 log 看到 200 OK 就以为成功（**假阳性**）

## 工作流错误

| 错误的"验证" | 为什么无效 |
|--------------|----------|
| 跑 pytest 全部通过 | 单元测试只用 mock LLM，不发真实请求 |
| `tsc --noEmit` 0 错 | 只查类型不查运行时 |
| 服务端 200 OK | 200 + 空 body = 看起来成功实际 NameError |
| "我跑了 4 个测试都过了" | 4 个 mock 测试 ≠ 端到端验证 |
