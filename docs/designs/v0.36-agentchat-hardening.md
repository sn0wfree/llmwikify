# v0.36 — AgentChat 全面硬化

**Status:** 计划 + 实施中
**Branch:** `feat/v0.36-agentchat-hardening`
**Date:** 2026-06-09
**Scope:** AgentChat 功能从"能跑"提升到"可生产"

---

## 1. 背景

v0.32-v0.35 完成 3-Facade DB、5+1-Service 架构、PPT 删除、SPA 合并后,AgentChat 仍是脆弱的多层实现。本次聚焦:

- **行为正确性:** 工具结果不回传 LLM、_save_message 静默吞错、UUID 碰撞、闭包陈旧、重复 DB 实例
- **架构统一:** ChatBase 与 ChatService 两套并行,MemoryManager 接入但未使用,系统提示词仅 4 行
- **可靠性:** 已有 LLMRetryManager/DBRetryManager 未应用,无限速/超时/级联
- **前端体验:** 无 Abort、无 token reset、仅最后一条可重生成、SSE 无重连
- **测试缺口:** AgentChat.tsx 无单测,chat_sse.py 无路由单测

---

## 2. 范围与决策

| 维度 | 决策 |
|---|---|
| 修复范围 | 全面升级 (5 Bug + 架构合并 + MemoryManager + 前端 + 重试 + 单测) |
| 工具循环 | 移植 `ChatBase.ask_with_tools` 到 `ChatBase`,新增 `aask_with_tools()` |
| ChatBase/Service 关系 | 保留 ChatBase,加异步方法;ChatService 复用 |
| MemoryManager | 完整集成 (提示词+历史+上下文+检索) |
| 前端 | Abort + token reset + 任意消息重生成 + SSE 自动重连 |
| Token 预算 | 用现有 `TokenBudgetChecker` |
| SSE 重连 | 客户端 fetch + 退避重试 (3 次) |
| LLM provider | 暂不增加 |
| DB 连接 | 共享 `app_db` 单例,启用外键,补级联删除 |
| 任务边界 | **仅评估+计划+实施,不向后兼容旧行为** |

## 4. 实施阶段

### Phase 0 — 基线 & 文档 (本次提交)
- 记录基线指标

- 切到 `feat/v0.36-agentchat-hardening` 分支

### Phase 1 — 严重 Bug 修复 (5 个)

| # | Bug | 修复 |
|---|---|---|
| 1.1 | 工具结果不回传 LLM (`service.py:283-347`) | 在 `ChatBase` 新增 `aask_with_tools()`,实现 reason→action→observe 循环 |
| 1.2 | `_save_message` 静默吞错 (`service.py:609`) | 改 `except: pass` 为 `logger.error` + 计数 + SSE warning 事件 |
| 1.3 | UUID 8 位碰撞 (`service.py:602`, `db.py:383`) | 用 `uuid.uuid4().hex` (32位) + `INSERT OR IGNORE` 替代 REPLACE |
| 1.4 | 前端闭包陈旧 (`AgentChat.tsx:180,259`) | 用 `useRef` 替代 `useState` 闭包 |
| 1.5 | 重复 DB 实例 (`agent_service.py:60` vs `service.py:221`) | `AgentService` 创建单一 `chat_db` 传给 `ChatService` |

### Phase 2 — 架构合并
- `ChatBase.aask_with_tools()` 完整实现
- `ChatService.chat()` 改为调用基类
- 删除 `service.py` 中重复的 `_stream_llm`/`_dispatch_tool_call` 循环部分

### Phase 3 — MemoryManager 完整集成
- 3.1 系统提示词注入 (用户偏好+工具列表+当前日期+Wiki 上下文)
- 3.2 历史恢复走 `memory.conversation.alist()`
- 3.3 工具结果存到 `context_entries`
- 3.4 `memory.index.search()` 检索相关历史注入提示词
- 3.5 异步化 `MemoryManager` (alist/aadd 包装,asyncio.to_thread)

### Phase 4 — 可靠性
- 4.1 `LLMRetryManager` 加 `aexecute()` 异步版本,接入 chat 流
- 4.2 `DBRetryManager` 加 `acall()`,接入 `_save_message`
- 4.3 速率限制中间件 (60 req/min/IP,token bucket)
- 4.4 SSE 心跳 15s + 总超时 5min
- 4.5 `delete_chat_session` 级联删除,启用外键

### Phase 5 — 前端体验
- 5.1 Abort 按钮 (StopCircle) + `AbortController` 透传 SSE
- 5.2 token 计数按 session 重置
- 5.3 任意消息可重生成 (新端点 `POST /api/agent/sessions/{id}/regenerate`)
- 5.4 SSE 自动重连 (退避 1s/2s/4s,最多 3 次,`Last-Event-ID` 续传)

### Phase 6 — 测试 & 文档
- 新增 6 个单测文件
- 更新 `docs/api/agent.md`
- 冒烟测试

## 6. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 工具循环无限 | 严格 `max_iterations=8` + `confirmation_required` 立即 break + 检测重复工具调用 |
| `aask_with_tools` 改动大 | 保留旧 `ask_with_tools` (同步) 供对比测试 |
| MemoryManager 异步化破坏现有 | 先加 alist/aadd 包装,再让 ChatService 迁移 |
| SSE 重连导致重复事件 | 服务端按 `last_event_id` 去重 |
| 速率限制误伤 | 默认 60 req/min 宽松,环境变量可关 |
| UUID 长度变化 | 新列+旧列共存,migration 不需要 |
