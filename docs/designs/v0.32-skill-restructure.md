# v0.32 Skill 重构 Design Doc

> **Sprint v0.32 of the 4-layer refactor series.**
> Per-session discussions 2026-06-08 distilled into a single actionable design document.
> Supersedes the `apps/agent/` deprecation mark from v0.30 and the 5901 LOC agent/ + 3442 LOC research/ + 3038 LOC ppt/ sprawl.

## 1. 背景与目标

### 1.1 现状问题

| 维度 | 现状数据 |
|---|---|
| `apps/agent/` | 21 文件 **5901 LOC**（WikiAgent runner + 20+ tools + memory + notify + scheduler + providers + hooks + dream_editor + routes）|
| `apps/research/` | 15 文件 **3442 LOC**（6 步研究子模块独立存在）|
| `apps/ppt/` | 10 文件 **3038 LOC**（PPT 生成独立存在）|
| `apps/chat/` | 31 文件 **7851 LOC**（6 步研究引擎 + chat 框架 + harness 三合一）|
| `apps/` 总占比 | **51.9%**（20251 / 38994 LOC）|

**核心问题**：
- **`apps/agent/` 是"什么都干"的巨型包**（同时承载 WikiAgent runner / 20+ tools / memory / notify / scheduler / providers / hooks / dream_editor / routes / db）
- **`apps/chat/` 混杂**（6 步研究子模块 + chat 框架 + eval harness）
- **`apps/research/` 与 `apps/chat/engine.py` 高度重叠**（同 6 步研究逻辑两边各一份）
- **`apps/ppt/` 与 chat 集成**（chat_router/chat_engine/chat_routes 三件套 793 LOC）
- **apps 内 4 个子包关系模糊**（chat 是中心？research/ppt 是被调模块？）

### 1.2 目标

1. **业务域清晰化**：apps 内 5 个业务域（wiki/chat/research/ppt/skills）边界明确
2. **Skill 化**：每个能力单元独立可测、可调、可复用
3. **Unix 哲学**：底层 12 skill = 工具池，上层 9 skill = 业务流程（详见 `docs/designs/principles/unix-philosophy.md`）
4. **Orchestrator = 上层 skill 的别名**：不重复实现
5. **MCP 零破坏**：28 个 wiki_* tool 派生自 wiki_orchestrator skill，外部客户端不受影响
6. **PPT 暂缓**：apps/ppt/ 21 文件保持现状，后续可能重写
7. **Tauri 桌面化**：v0.32 完成后立即启动 v0.33.0
8. **v0.32 分两阶段**：v0.32.0（核心 1.5 月）+ v0.32.5（完整 1 月）

### 1.3 非目标

- 不做 Tauri 桌面化（v0.32 完成后立即做，但不在 v0.32 范围）
- 不做 i18next 多语言
- 不迁移到 Milkdown 编辑器
- 不重写 apps/ppt/（PPT 暂缓，后续可能重写）
- 不完全 DDD 化（采用方案 1 而非方案 2）

## 3. Skill 体系（Unix 哲学）

详见 `docs/designs/principles/unix-philosophy.md`。3 条核心原则：
1. **底层 = 工具**（单一职责、纯数据操作）
2. **上层 = 业务流程**（编排底层完成业务目标）
3. **简单 CRUD 不拆**（已经是 Unix 风格）

### 3.3 上层 skill 调底层 skill 的内部机制

```python
# apps/chat/skills/gather_skill.py
class GatherSkill(Skill):
    name = "gather"
    actions = {
        "gather_for_research": SkillAction(handler=gather_for_research, ...),
    }

async def gather_for_research(args, ctx):
    """业务流程：search → filter → extract."""
    # 1. 调底层 search_skill
    search_result = await search_skill.actions["search"].handler(
        {"query": args["query"], "limit": 50}, ctx
    )

    # 2. 调底层 filter_skill
    filter_result = await filter_skill.actions["filter"].handler(
        {"sources": search_result.data["results"], "min_score": 0.3}, ctx
    )

    # 3. 调底层 extract_skill
    sources = []
    for s in filter_result.data["filtered"][:args.get("max_sources", 20)]:
        extract = await extract_skill.actions["extract"].handler(
            {"url_or_path": s["url"]}, ctx
        )
        sources.append({**s, "content": extract.data["content"]})

    return SkillResult(status="ok", data={"sources": sources})


# apps/chat/skills/research_skill.py
async def run_research(args, ctx):
    """6 步研究：plan → gather → analyze → summarize → score → revise → report."""
    state = ResearchState()

    # 步 1: plan (底层)
    state.plan = await plan_skill.actions["plan"].handler(args, ctx)

    # 步 2: gather (上层, 内部调 search+filter+extract)
    state.sources = await gather_skill.actions["gather_for_research"].handler(
        {"query": args["query"], "sub_queries": state.plan.data["sub_queries"]}, ctx
    )

    # 步 3: analyze (底层)
    state.analyzed = await analyze_skill.actions["analyze"].handler(..., ctx)

    # 步 4: summarize (底层)
    state.summary = await summarize_skill.actions["summarize"].handler(..., ctx)

    # 步 5: score (底层)
    score = await score_skill.actions["score"].handler(..., ctx)

    # 步 5b: revise (底层) 如 score < 阈值
    if score.data["score"] < ctx.config.get("quality_threshold", 7):
        state.summary = await revise_skill.actions["revise"].handler(..., ctx)

    # 步 6: report (上层, 内部调 summarize+write+score)
    final = await report_skill.actions["generate_report"].handler(..., ctx)

    return SkillResult(status="ok", data={"report": final.data, "state": state.to_dict()})
```

**关键设计点**：
- **上层 skill 调底层 skill** 通过 Python 函数调用（**不通过 LLM**）
- 状态机 6 步全部由 Python 驱动
- 状态持久化在 `research_skill.run_research` 内部
- LLM 可选粒度：高层一键 vs 底层组合

## 6. MCP 28 tool 派生（零外部破坏）

### 6.1 派生机制

```python
# interfaces/mcp/tools.py (NEW)
def register_skill_tools(mcp, registry):
    for skill in registry.skills.values():
        for action_name, action in skill.actions.items():
            @mcp.tool
            def {action_name}(**kwargs):
                return action.handler(kwargs, ...)
```

### 6.2 零破坏保证
- 28 个 tool **name 不变**：`read_page`, `write_page`, `search`, ...
- 28 个 tool **描述字符串**：`wiki_query_skill.actions["read_page"].description`（**显式 hardcode 原文**，不从 manifest 重新生成）

- 测试：`assert registered_tool.description == original_description`（string equality）

## 17. Phase 0 调研报告（2026-06-08）

**目标**：通读 19 skill 对应的源文件，确认合并可行性，识别隐藏依赖与工期调整。

### 17.1 关键发现 1：`apps/research/engine.py` 是 **ReAct 循环**而非简单 6 步

**之前设计假设**：`research_skill.run_research` = 6 步顺序（plan → gather → analyze → synthesize → review → report）

**实际发现**（读 `engine.py:142-1178` 后）：
- `ResearchEngine` 实现的是 **ReAct 自适应循环**（Reason → Act → Observe → loop）
- 含 `VALID_TRANSITIONS` 转移表（line 34-45）
- 8 个 `_action_*` 方法：plan / gather / analyze / synthesize / report / **review** / **revise** / done（不是 6 步，是 7 个有效 action）
- LLM 在 `_llm_reason` 步骤（line 436-501）**动态决定**下一步（基于 state.observations）
- 状态机含 `_evaluate_gate` 干预（line 595-606）—— gate 失败会强制改 action
- 状态机含 `_check_control_signals`（line 505-516）—— DB 中的 cancel/pause 信号
- `_observe`（line 520-591）从 DB 重新读 state 并生成 5+ 类可读 observation

**对 plan 的影响**：

| 维度 | 原假设 | 实际 |
|---|---|---|
| `research_skill` 内部 | 6 步顺序 | **ReAct 循环 + LLM reason + 7 步 action 转移** |
| 状态机 | 简单 6 步 | 8 个状态 + VALID_TRANSITIONS + gate 干预 + LLM reason |
| 持久化字段 | 6 个 step 结果 | **15+ 字段**（state.phase/round/sources/sub_queries/synthesis/report_md/review/knowledge_gaps/contradictions/issues/observations/_last_thought/cancelled/paused/budget_remaining）|
| 工期 | 中 | **高**（ReAct 复杂 + 持久化 + LLM reason 状态保存）|

**修订**：`research_skill.run_research` 必须**保留 ReAct 循环结构**——`_reason` 内部 LLM 调一次（line 436），不是多次 LLM 调用。整体逻辑从 `ResearchEngine` 类**整体迁移**到 `apps/chat/skills/research_skill.py`，7 个 `_action_*` 方法**逐一映射**到底层/上层 skill：

| `_action_analyze` | `analyze_skill.analyze` | 底层 |
| `_action_synthesize` | `summarize_skill.summesize`（拼写按 skill）| 底层 |

| `_action_revise` | `revise_skill.revise` | 底层 |

async def observe_research_state(args, ctx):
    """从 state + DB 生成 5+ 类 observations (纯函数)."""
    sources = args.get("sources", [])
    sub_queries = args.get("sub_queries", [])
    synthesis = args.get("synthesis")
    observations = []

    # 平均可信度
    analyzed = [s for s in sources if s.get("analysis")]
    if analyzed:
        cred_scores = [s.get("analysis", {}).get("quality_assessment", {}).get("credibility", 5) for s in analyzed]
        avg_cred = sum(cred_scores) / len(cred_scores) if cred_scores else 0
        observations.append(
            f"Average source credibility: {avg_cred:.1f}/10 ({len(analyzed)}/{len(sources)} analyzed)"
        )

    # 失败 sub-queries
    failed = [sq for sq in sub_queries if sq.get("status") == "failed"]
    if failed:
        observations.append(f"{len(failed)} sub-queries failed: {[sq['query'] for sq in failed[:3]]}")

    # 来源类型分布
    type_counts = {}
    for s in sources:
        t = s.get("source_type", "unknown")
        type_counts[t] = type_counts.get(t, 0) + 1
    if type_counts:
        observations.append(f"Source types: {type_counts}")

    # Local wiki vs web ratio
    wiki_count = type_counts.get("wiki", 0)
    web_count = type_counts.get("web", 0)
    if wiki_count + web_count > 0:
        observations.append(f"Local wiki: {wiki_count} sources, Web: {web_count} sources")

    # 知识缺口
    if synthesis:
        gaps = synthesis.get("knowledge_gaps", [])
        if len(gaps) > 3:
            observations.append(f"⚠ {len(gaps)} knowledge gaps may impact report completeness")

    return SkillResult(status="ok", data={"observations": observations})

# apps/chat/skills/reason_skill.py
class ReasonSkill(Skill):
    """ReAct 推理: 决定下一步 action.

    复用方:
    - research_skill._react_loop (every round)
    - harness 评测 (golden case 中手动 step-through)
    - 自定义 agent (ReAct 风格研究)
    """
    name = "reason"
    actions = {
        "reason_research": SkillAction(
            description="Decide next research action (LLM-driven with rule-based fallback)",
            input_schema={
                "type": "object",
                "properties": {
                    "state": {"type": "object", "description": "ResearchState dict"},
                },
                "required": ["state"],
            },
            handler=reason_research,
        ),
    }


async def reason_research(args, ctx):
    """ReAct 推理: LLM 决定下一步, fallback 到 rule-based."""
    state = args["state"]

    # 1. 调 LLM reason
    try:
        return await _llm_reason(state, ctx)
    except Exception as e:
        logger.warning(f"LLM reason failed: {e}, using rule-based fallback")

    # 2. Rule-based fallback
    result = _rule_based_reason(state) or {"action": "done", "thought": "fallback exhausted"}
    return SkillResult(status="ok", data=result)


async def _llm_reason(state, ctx):
    """LLM-driven reason: 发送 messages, 解析 JSON 返回."""
    import asyncio
    import json as json_mod

    analyzed_count = sum(1 for s in state.get("sources", []) if s.get("analysis"))
    failed_sq = sum(1 for sq in state.get("sub_queries", []) if sq.get("status") == "failed")
    obs_text = "\n".join(f"  - {o}" for o in state.get("observations", [])) or "  (none)"

    messages = [
        {"role": "system", "content": (
            "You are a research orchestrator using ReAct reasoning.\n"
            'Return a JSON object with:\n'
            '- "thought": Your reasoning (1-2 sentences)\n'
            '- "action": One of: plan, gather, analyze, synthesize, report, review, revise, done\n\n'
            "Rules:\n"
            "- If sub-queries exist but not all gathered → gather\n"
            "- If sources exist but not all analyzed → analyze\n"
            "- If all analyzed but no synthesis → synthesize\n"
            "- If synthesis shows knowledge gaps and budget allows → plan (replan)\n"
            "- If report exists but not reviewed → review\n"
            "- If review failed and rounds remain → revise\n"
            "- If all done and quality acceptable → done\n"
        )},
        {"role": "user", "content": (
            f"Research topic: {state.get('query', '')}\n"
            f"Round: {state.get('round', 0)}/{state.get('max_rounds', 5)}\n"
            f"Phase: {state.get('phase') or 'starting'}\n"
            f"Quality score: {state.get('quality_score', 0)}/10\n"
            f"Budget remaining: {state.get('budget_remaining', 1.0):.0%}\n\n"
            f"Sub-queries: {len(state.get('sub_queries', []))} ({failed_sq} failed)\n"
            f"Sources gathered: {len(state.get('sources', []))}\n"
            f"Sources analyzed: {analyzed_count}\n"
            f"Report exists: {bool(state.get('report_md'))}\n"
            f"Review exists: {bool(state.get('review'))}\n\n"
            f"Observations:\n{obs_text}\n\n"
            "What should I do next? Return JSON."
        )},
    ]

    raw = await asyncio.to_thread(ctx.llm_client.chat, messages, max_tokens=200, temperature=0.1)
    raw = raw.strip()
    if raw.startswith("```"):
        raw = raw.split("\n", 1)[1] if "\n" in raw else raw[3:]
        if raw.endswith("```"):
            raw = raw[:-3]
        raw = raw.strip()

    result = json_mod.loads(raw)
    action = result.get("action", "done")
    thought = result.get("thought", "")

    valid = {"plan", "gather", "analyze", "synthesize", "report", "review", "revise", "done"}
    if action not in valid:
        action = "done"

    return SkillResult(status="ok", data={"action": action, "thought": thought})


def _rule_based_reason(state):
    """Rule-based fallback: 确定性状态转移决策 (从 engine.py:382-434 抽出)."""
    phase = state.get("phase", "")

    if phase == "error":
        return {"action": "done", "thought": "error state, finishing"}
    if not state.get("sub_queries"):
        return {"action": "plan", "thought": "no sub-queries, need planning"}

    gathered_ids = {s.get("sub_query_id") for s in state.get("sources", [])}
    ungathered = [sq for sq in state["sub_queries"] if sq["id"] not in gathered_ids and sq.get("status") != "failed"]
    if ungathered:
        return {"action": "gather", "thought": f"{len(ungathered)} ungathered sub-queries"}

    unanalyzed = [s for s in state.get("sources", []) if not s.get("analysis")]
    if unanalyzed:
        return {"action": "analyze", "thought": f"{len(unanalyzed)} unanalyzed sources"}

    if state.get("synthesis") is None:
        return {"action": "synthesize", "thought": "no synthesis yet"}

    if (state.get("knowledge_gaps")
            and state.get("budget_remaining", 1.0) > 0.15
            and state.get("round", 0) < state.get("max_replan", 2) + 1):
        return {"action": "plan", "thought": "knowledge gaps, replanning"}

    if state.get("report_md") is None:
        return {"action": "report", "thought": "no report yet"}
    if state.get("review") is None:
        return {"action": "review", "thought": "report not reviewed"}
    if state.get("review", {}).get("approved"):
        return {"action": "done", "thought": "review approved"}
    if state.get("round", 0) < state.get("max_rounds", 5):
        return {"action": "revise", "thought": "review failed, revising"}
    return {"action": "done", "thought": "max rounds reached"}

### 19.3 ReactLoop 框架详细设计

```python
# apps/chat/agent/react_loop.py
"""通用 ReAct 循环框架.

不绑定具体业务. 任何 ReAct 实例（研究/数据分析/代码生成/客户支持）
都用此框架 + 不同的 actions + 不同的 reason_prompt + 不同的 hooks.
"""

from __future__ import annotations

import logging
from collections.abc import AsyncIterator
from dataclasses import dataclass, field
from typing import Any, Callable

from llmwikify.apps.chat.skills.base import Skill, SkillAction, SkillContext
from llmwikify.apps.chat.skills.reason_skill import reason_skill
from llmwikify.apps.chat.skills.observe_skill import observe_skill

logger = logging.getLogger(__name__)


@dataclass
class ReactConfig:
    """ReactLoop 完整配置. 9 个字段均可自定义."""
    # ── 必填 ──
    actions: list[SkillAction]                 # 可用 actions 列表
    initial_state: dict                        # 初始 state
    reason_prompt: str                         # LLM prompt 模板
    done_condition: Callable[[dict], bool]     # 终止条件

    # ── 可选（高度参数化）──
    max_rounds: int = 10                       # 最大循环轮数
    observation_skill: Skill = observe_skill   # 提供 observations 的 skill
    reason_skill: Skill = reason_skill         # 提供 reason 的 skill
    on_before_act: Callable | None = None       # hook: action 执行前
    on_after_act: Callable | None = None        # hook: action 执行后 (gate 干预)
    on_before_observe: Callable | None = None   # hook: observe 前
    on_after_observe: Callable | None = None    # hook: observe 后
    persist_state: Callable | None = None      # hook: 每 round 后持久化
    restore_state: Callable | None = None       # hook: 启动时恢复


class ReactLoop:
    """通用 ReAct 循环: Reason → Act → Observe + 4 hooks."""

    def __init__(self, config: ReactConfig):
        self.config = config
        self.state = dict(config.initial_state)
        self.action_map = {a.name: a for a in config.actions}

    async def run(self, ctx: SkillContext) -> AsyncIterator[dict[str, Any]]:
        """主循环. 每次 round: REASON → ACT → OBSERVE + hooks."""
        # Hook: 恢复 state (用于 resume 场景)
        if self.config.restore_state:
            self.state = self.config.restore_state(self.state)

        for round in range(self.config.max_rounds):
            # ── 终止检查 ──
            if self.config.done_condition(self.state):
                yield {"type": "phase", "phase": "done", "round": round}
                break

            # ── REASON: 调 reason_skill ──
            reason_result = await self.config.reason_skill.actions["reason_research"].handler(
                {"state": self.state}, ctx
            )
            action_name = reason_result.data["action"]
            thought = reason_result.data["thought"]
            yield {"type": "reasoning", "action": action_name, "thought": thought, "round": round}

            # ── HOOK: on_before_act ──
            if self.config.on_before_act:
                self.config.on_before_act(self.state, action_name)

            # ── ACT: 调对应 action handler ──
            action = self.action_map.get(action_name)
            if action is None:
                logger.warning(f"Unknown action {action_name}, skipping")
                continue
            try:
                result = await action.handler(self.state, ctx)
                # action 返回的 data 更新 state
                if result.status == "ok" and result.data:
                    self.state.update(result.data)
            except Exception as e:
                logger.error(f"Action {action_name} failed: {e}", exc_info=True)
                yield {"type": "action_error", "action": action_name, "error": str(e)}
                continue

            # ── HOOK: on_after_act (gate 干预等) ──
            if self.config.on_after_act:
                self.config.on_after_act(self.state, action_name, result)

            # ── HOOK: on_before_observe ──
            if self.config.on_before_observe:
                self.config.on_before_observe(self.state)

            # ── OBSERVE: 调 observe_skill ──
            observe_result = await self.config.observation_skill.actions["observe_research_state"].handler(
                {
                    "session_id": self.state.get("session_id"),
                    "sources": self.state.get("sources", []),
                    "sub_queries": self.state.get("sub_queries", []),
                    "synthesis": self.state.get("synthesis"),
                },
                ctx
            )
            self.state["observations"] = observe_result.data["observations"]

            # ── HOOK: on_after_observe ──
            if self.config.on_after_observe:
                self.config.on_after_observe(self.state)

            # ── HOOK: persist_state ──
            if self.config.persist_state:
                self.config.persist_state(self.state, round)

            yield {
                "type": "round_complete",
                "round": round,
                "action": action_name,
                "state_snapshot": dict(self.state),
            }

        yield {"type": "phase", "phase": "done", "final_state": self.state}
```

### 19.4 research_skill 重构（用 ReactLoop 配置化）

```python
# apps/chat/skills/research_skill.py
"""研究流程 = ReactLoop + 8 research actions + research hooks.
总 LOC: ~80 行 (vs. 重写 ReAct 循环的 ~180 行).
"""

from __future__ import annotations

import logging
from typing import Any

from ..agent.react_loop import ReactLoop, ReactConfig
from .base import Skill, SkillAction, SkillContext
from .plan_skill import plan_skill
from .gather_skill import gather_skill
from .analyze_skill import analyze_skill
from .summarize_skill import summarize_skill
from .score_skill import score_skill
from .revise_skill import revise_skill
from .report_skill import report_skill

logger = logging.getLogger(__name__)


class ResearchSkill(Skill):
    name = "research"
    actions = {
        "run_research":   SkillAction(handler=run_research, ...),
        "resume_research": SkillAction(handler=resume_research, ...),
        "cancel_research": SkillAction(handler=cancel_research, ...),
    }


# Research-specific reason prompt
RESEARCH_REASON_PROMPT = """\
You are a research orchestrator using ReAct reasoning.
Based on the current state, decide the next action.

Return JSON:
- "thought": reasoning (1-2 sentences)
- "action": one of: plan, gather, analyze, synthesize, report, review, revise, done

Rules:
- If sub-queries exist but not all gathered → gather
- If sources exist but not all analyzed → analyze
- If all analyzed but no synthesis → synthesize
- If synthesis shows knowledge gaps and budget allows → plan (replan)
- If report exists but not reviewed → review
- If review failed and rounds remain → revise
- If all done and quality acceptable → done
"""


async def run_research(args: dict, ctx: SkillContext):
    """研究流程入口. 配置化 ReactLoop 实例化."""
    config = _make_research_config(args)
    loop = ReactLoop(config)
    return await loop.run(ctx)


def _make_research_config(args: dict) -> ReactConfig:
    """构造研究流程的 ReactConfig."""

    # ── 8 个 research-specific actions ──
    actions = [
        plan_skill.actions["plan"],
        gather_skill.actions["gather_for_research"],
        analyze_skill.actions["analyze"],
        summarize_skill.actions["summarize"],
        score_skill.actions["score"],
        revise_skill.actions["revise"],
        report_skill.actions["generate_report"],
    ]

    # ── research-specific hooks ──
    def check_control_signals(state, action_name):
        """Hook: 检查 DB cancel/pause 信号."""
        session = db.get_research_session(state.get("session_id"))
        if session:
            if session.get("status") == "cancelling":
                state["cancelled"] = True
            elif session.get("status") == "pausing":
                state["paused"] = True

    def gate_intervention(state, action_name, result):
        """Hook: gate 评估干预 (研究专用)."""
        gate = _evaluate_gate(state)
        if gate and not gate.passed:
            state["observations"].append(
                f"⚠ Gate {gate.gate_name} failed: {gate.summary}"
            )
            # 强制改下次 action: gathering 失败 → analyze
            if state.get("phase") == "gathering" and len(state.get("sources", [])) >= state.get("gate_min_sources", 3):
                state["_forced_next_action"] = "analyze"

    def persist_research_state(state, round):
        """Hook: 持久化 state (research 专用)."""
        save_research_state_to_db(state, round)

    def restore_research_state(state):
        """Hook: 恢复 state (用于 resume 场景)."""
        return load_research_state_from_db(state.get("session_id"))

    # ── 初始 state ──
    initial_state = {
        "session_id": args.get("session_id"),
        "query": args.get("query"),
        "round": 0,
        "max_rounds": args.get("max_rounds", 5),
        "max_replan": args.get("max_replan", 2),
        "phase": "",
        "sub_queries": [],
        "sources": [],
        "synthesis": None,
        "report_md": None,
        "review": None,
        "knowledge_gaps": [],
        "contradictions": [],
        "issues": [],
        "observations": [],
        "cancelled": False,
        "paused": False,
        "budget_remaining": 1.0,
        "_last_thought": "",
    }

    return ReactConfig(
        actions=actions,
        initial_state=initial_state,
        reason_prompt=RESEARCH_REASON_PROMPT,
        done_condition=lambda s: (
            s.get("phase") == "done"
            or s.get("cancelled")
            or s.get("paused")
        ),
        max_rounds=args.get("max_rounds", 5),
        on_before_act=check_control_signals,
        on_after_act=gate_intervention,
        persist_state=persist_research_state,
        restore_state=restore_research_state,
    )


async def resume_research(args: dict, ctx: SkillContext):
    """从中断点恢复研究. 复用 _make_research_config + restore_state hook."""
    args["resume"] = True
    return await run_research(args, ctx)


async def cancel_research(args: dict, ctx: SkillContext):
    """取消研究. 更新 DB 状态为 cancelling, 下次 _check_control_signals 触发."""
    db.update_research_status(args["session_id"], "cancelling", "cancelling", -1)
    return {"status": "cancelled"}


def _evaluate_gate(state):
    """研究专用: 4 个 gate 评估 (check_after_gathering/analysis/.../before_report)."""
    gate = state.get("_quality_gate")  # 在 initial_state 中注入
    phase = state.get("phase", "")
    if phase == "gathering":
        return gate.check_after_gathering(state.get("sources", []), state.get("sub_queries", []))
    elif phase == "analyzing":
        return gate.check_after_analysis(state.get("sources", []))
    elif phase == "synthesizing":
        return gate.check_after_synthesis(state.get("synthesis"))
    elif phase == "reporting":
        return gate.check_before_report(state.get("synthesis"), state.get("sources", []))
    return None
```

**`research_skill` 文件总 LOC**：~180 → **~100 行**（−44%）
