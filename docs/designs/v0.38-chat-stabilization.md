# v0.38 — Chat 功能清理与稳定化

**Status:** 执行中
**Branch:** `feat/v0.36-agentchat-hardening`
**Date:** 2026-06-10
**Scope:** 删除死代码、合并重复实现、统一 ReAct 引擎、补异常日志、可配置化

---

## 1. 背景

v0.37 统一 ReAct 循环后，代码层仍存在以下问题：

| 问题 | 严重程度 | 影响 |
|------|----------|------|
| `service.py` 本地文本解析器与 `text_mode_tool.py` 重复 | 中 | 两份实现漂移 |
| `base.py` 同步 API（`ask`/`ask_with_tools`/`invoke_tool`）零调用方 | 低 | 认知负担 |
| `AgentContext._recent_tool_entries`/`tool_invocations` 只写不读 | 低 | 内存泄漏 |
| `use_react_engine=False` fallback 路径 ~110 行死代码 | 中 | 维护负担 |
| 魔术数字散落 4+ 文件 | 中 | 配置困难 |
| 异常吞咽无日志（`except Exception: return None` ×3） | 高 | 不可观测 |
| `ReactLoop`（legacy）仍独立存在于 `react_loop.py` | 中 | 三实现未统一 |
| 前端 `ChatStreamEvent` 类型撒谎（`actions: unknown[]`） | 低 | 类型不可信 |

---

## 2. 目标

1. **删除死代码**：零调用方函数、只写不读字段、重复实现
2. **合并重复**：文本解析器统一到 `text_mode_tool.py`
3. **可配置化**：魔术数字集中到 `config.py`
4. **异常可观测**：所有 `except Exception: return None` 补日志
5. **路径合并**：删除 `use_react_engine=False` fallback
6. **ReAct 统一**：`react_loop.py` 改为 re-export wrapper
7. **前端修复**：类型签名与后端对齐

---
