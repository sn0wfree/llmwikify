# v0.37 — Triple ReAct Loop 统一

**Status:** 设计中
**Branch:** `feat/v0.37-react-loop` (从 `feat/v0.32-skill-refactor` 切)
**Date:** 2026-06-09
**Scope:** 把 ChatService / ResearchEngine / AutoResearch 中分散的 ReAct 循环统一为 `ReActEngine`

---

## 1. 背景

v0.36 hardening 完成后,AgentChat 行为层达到可生产质量。但代码层暴露了一个架构问题:**ReAct 循环分散在 3 处**,每处独立维护 reason → action → observe 三阶段:

| 位置 | 文件 | 步数 | 触发场景 |
|---|---|---|---|
| Framework 通用 | `framework/react.py` | 通用骨架 | Skill pipeline 调用 |
| ChatService | `apps/chat/agent/service.py` (隐式) | 6 步 (经 `aask_with_tools`) | 用户 chat |
| ResearchEngine | `apps/chat/engine.py: _react_loop` | 6 步 (ResearchEngine 状态机) | Quick Research |
| AutoResearchEngine | `apps/research/engine.py` | 7 步 | AutoResearch |

**重复代价:**
- 同样的 lifecycle hook (timeout / cancel / pause) 在 4 处独立实现
- thinking snapshot 写入策略分散
- confirmation_required 拦截逻辑在 ChatService 和 ResearchEngine 重复
- 单元测试覆盖无法共享,新增 lifecycle hook 需改 4 处

---

## 2. 目标

把 4 处 ReAct 实现统一到 **1 个 `ReActEngine`** + **2 个适配层** (ChatReActBridge + ResearchEngineAdapter):

```
                ┌─────────────────────┐
                │   ReActEngine       │
                │   (685L, 13 步)     │
                │   7 lifecycle hooks │
                └──────────┬──────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ChatReActBridge    _build_react_config  (Direct)
   (ChatService)      (ResearchEngine)     (framework.react)
        │                  │                  │
   ChatService        ResearchEngine      Skill pipelines
```

### 2.1 ReActEngine 核心契约 (13 步 round)

每轮迭代按以下顺序触发:

```
1.  pre_reason       (可注入:上下文增强)
2.  reason           (LLM 调用)
3.  post_reason      (可注入:解析 thinking)
4.  pre_action       (可注入:action 校验)
5.  action           (执行工具/函数)
6.  post_action      (可注入:结果包装)
7.  pre_observe      (可注入:observation 拼装)
8.  observe          (生成 observation)
9.  post_observe      (可注入:持久化)
10. pre_decide        (可注入:继续/终止判断)
11. decide            (终止决策:done / continue / fail)
12. post_decide       (可注入:状态写回)
13. on_round_complete (可注入:round 结束 hook)
```

### 2.2 Lifecycle Hooks (7 个)

| Hook | 用途 |
|---|---|
| `on_session_start` | session 初始化 |
| `on_round_start` | 每轮 round 开头 |
| `on_round_complete` | 每轮 round 结尾 |
| `on_session_complete` | session 终止 (done) |
| `on_error` | 异常捕获 |
| `on_timeout` | 总超时 (默认 300s) |
| `on_cancel` | AbortSignal 触发 |

### 2.3 事件类型

```python
EVENT_REASONING = "reasoning"           # LLM 流式文本
EVENT_ACTION = "action"                 # 工具调用执行
EVENT_OBSERVATION = "observation"       # 工具结果
EVENT_ROUND_COMPLETE = "round_complete" # 每轮结束
EVENT_ACTION_ERROR = "action_error"     # 工具调用失败
EVENT_OBSERVATION_ERROR = "observation_error"
EVENT_PHASE = "phase"                   # ResearchEngine 域事件
EVENT_TIMEOUT = "timeout"
EVENT_DONE = "done"
```

## 3. 范围与决策

| 维度 | 决策 |
|---|---|
| 统一粒度 | 全部 ReAct 实现走 ReActEngine |
| ChatService 默认路径 | `use_react_engine=True` 默认开启 |
| Fallback 路径 | 保留 `aask_with_tools` 经 `use_react_engine=False` (回归测试) |
| ResearchEngine 集成 | 通过 `_build_react_config` 把 6 步映射为 13 步通用 round |
| AutoResearch | 通过同样的 config adapter |
| Lifecycle hooks | 7 个全部实现,可选注入 |
| Timeout | 默认 300s, 可经 `ReActConfig.timeout_s` 覆盖 |
| Cancel | 经 `AbortSignal` / `CancelToken` |
| Thinking snapshot | `ReActEngine` 维护 `_thinking: str`, 每轮 reason 后更新 |
| Confirmation 拦截 | 在 `decide` step 内检测 `confirmation_required`, 立即 break |

### 3.1 不在范围

- ReActEngine 不做工具发现 (仍由调用方传入 `SkillAction` 列表)
- ReActEngine 不做 DB 持久化 (由调用方在 hooks 内实现)
- ReActEngine 不做 SSE 事件流 (由 ChatReActBridge 翻译)

## 6. 风险与缓解

| 风险 | 缓解 |
|---|---|
| ReActEngine 与 aask_with_tools 双路径并存 | 默认 `use_react_engine=True`, 保留 fallback 参数; 测试覆盖两条路径 |
| ResearchEngine 6 步 → 13 步 round 映射损失语义 | 通过 `_build_react_config` 显式映射, 6 个 phase 注入 `EVENT_PHASE` |
| 大量新增 lifecycle hook 引入 bug | ReActEngine 单元测试覆盖 13 步 × 7 hook = 91 个组合点 (实际用 mock 覆盖关键路径) |
| ChatReActBridge 翻译事件遗漏 | 单元测试用 golden 事件流对比 |
| 真实 LLM 冒烟成本 | 6 场景用 gpt-4o-mini (低成本), 失败时记录场景原因 |
| 73fd230 LLM error body 兼容 | bridge 在 retry 时透传 error message |
